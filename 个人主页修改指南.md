# Not Pure Poole 模板说明与个人主页修改指南

## 一、文件/文件夹结构与作用

### 核心配置文件
1. **_config.yml** - 网站主配置文件
   - 包含网站标题、描述、URL、头像等基本信息
   - 配置插件、SEO设置、作者信息等

### 数据文件（_data/）
2. **_data/navigation.yml** - 导航菜单配置
   - 定义网站顶部导航栏的链接

3. **_data/social.yml** - 社交媒体链接配置
   - 定义网站底部或侧边栏的社交媒体图标和链接

4. **_data/archive.yml** - 归档页面配置
   - 配置按日期、分类和标签归档的页面链接

### 布局与模板（_layouts/ & _includes/）
5. **_layouts/** - 页面布局模板
   - default.html: 默认布局
   - home.html: 首页布局
   - page.html: 普通页面布局
   - post.html: 博客文章布局
   - archive-*.html: 归档页面布局

6. **_includes/** - 可复用HTML片段
   - head.html: 网页头部（元标签、样式表等）
   - home-header.html: 首页头部（封面图片、标题等）
   - content.html: 内容容器
   - sidebar-*.html: 侧边栏组件
   - disqus.html: Disqus评论组件
   - google-analytics.html: Google Analytics跟踪代码

### 样式文件（_sass/）
7. **_sass/** - Sass样式文件
   - _variables.scss: 变量定义（颜色、字体等）
   - _layout.scss: 布局样式
   - _type.scss: 排版样式
   - _sidebar.scss: 侧边栏样式
   - _posts.scss: 博客文章样式
   - _code.scss: 代码高亮样式

8. **assets/styles.scss** - 主样式文件
   - 导入所有Sass部分文件并生成最终CSS

### 内容文件
9. **index.html** - 网站首页
   - 使用home.html布局

10. **about.md** - 关于页面
    - 使用page.html布局

11. **_posts/** - 博客文章文件夹
    - 包含所有博客文章，文件名格式：YYYY-MM-DD-标题.md

### 其他文件
12. **Gemfile** & **Gemfile.lock** - Ruby依赖管理文件
    - 定义Jekyll和主题所需的Ruby gems

13. **README.md** - 主题说明文档
    - 包含主题特性、安装和使用说明

14. **avatar.jpeg** & **bg.jpeg** - 示例图片文件
    - 头像和封面背景图片

## 二、个人主页修改步骤

### 1. 基本信息修改

**修改文件：_config.yml**

```yaml
# 网站基本信息
title: 你的名字或网站名称
description: 你的网站描述
url: https://yourusername.github.io  # 你的GitHub Pages URL
avatar: /avatar.jpeg  # 头像图片路径
cover_image: /bg.jpeg  # 封面图片路径

# 作者信息
author:
  name: 你的名字
  url: https://github.com/yourusername/  # 你的GitHub或个人网站
  email: your@email.com  # 你的邮箱

# 社交媒体链接（也可以在_data/social.yml中修改）
social:
  name: 你的名字
  links:
    - https://twitter.com/yourusername
    - https://github.com/yourusername

# 语言设置
lang: zh-CN  # 设置为中文
```

### 2. 导航菜单修改

**修改文件：_data/navigation.yml**

```yaml
- title: 首页
  url: /
- title: 关于
  url: /about/
- title: 博客
  url: /
- title: 项目
  url: /projects/  # 如果你创建了项目页面
- title: 联系方式
  url: /contact/  # 如果你创建了联系方式页面
```

### 3. 社交媒体链接修改

**修改文件：_data/social.yml**

```yaml
- title: 邮箱
  url: mailto://your@email.com
  icon: fas fa-envelope
- title: GitHub
  url: https://github.com/yourusername
  icon: fab fa-github
- title: 微博
  url: https://weibo.com/yourusername
  icon: fab fa-weibo
- title: 知乎
  url: https://www.zhihu.com/people/yourusername
  icon: fab fa-zhihu  # 注意：Font Awesome可能没有知乎官方图标，你可能需要使用其他图标
- title: LinkedIn
  url: https://www.linkedin.com/in/yourusername/
  icon: fab fa-linkedin
```

### 4. 关于页面修改

**修改文件：about.md**

```markdown
---
layout: page
title: 关于我
---

这里是你的个人介绍内容。你可以写：
- 你的背景和经历
- 你的技能和专长
- 你的兴趣爱好
- 联系方式

使用Markdown语法可以轻松添加：
- **粗体**和*斜体*文本
- [链接](https://example.com)
- 列表项
- 图片
```

### 5. 替换图片

- 将你的头像图片命名为`avatar.jpeg`并替换根目录下的现有文件
- 将你的封面图片命名为`bg.jpeg`并替换根目录下的现有文件
- 如果你想使用不同的文件名，需要在`_config.yml`中更新相应的路径

### 6. 添加自定义页面

如果你想添加新页面（如项目、联系方式等）：

1. 在根目录下创建新的Markdown文件，如`projects.md`
2. 添加页面头部信息（YAML Front Matter）：

```markdown
---
layout: page
title: 我的项目
---

这里是你的项目内容...
```

3. 在`_data/navigation.yml`中添加对应的导航链接

### 7. 修改主题颜色和样式

如果你想自定义主题颜色：

**修改文件：_sass/_variables.scss**

```scss
// 主色调
--brand-color: #你的颜色值;

// 背景色
--body-bg: #你的背景色;

// 文本颜色
--body-color: #你的文本颜色;

// 链接颜色
--link-color: #你的链接颜色;
--link-hover-color: #你的链接悬停颜色;
```

### 8. 添加博客文章

1. 在`_posts/`文件夹中创建新的Markdown文件
2. 文件名格式必须为：`YYYY-MM-DD-文章标题.md`
3. 添加文章头部信息：

```markdown
---
layout: post
title: 文章标题
date: 2023-05-15 10:00:00
categories: [分类1, 分类2]
tags: [标签1, 标签2]
---

这里是文章内容...
```

## 三、本地测试

在修改完文件后，你可以在本地测试网站效果：

1. 确保已安装Ruby和Bundler
2. 打开命令行，进入项目根目录
3. 运行`bundle install`安装依赖
4. 运行`bundle exec jekyll serve`启动本地服务器
5. 在浏览器中访问`http://localhost:4000`查看效果

## 四、发布到GitHub Pages

1. 将项目推送到GitHub仓库（仓库名应为`yourusername.github.io`）
2. 在仓库设置中启用GitHub Pages
3. 等待几分钟，你的个人主页将在`https://yourusername.github.io`上可用

## 五、高级自定义

### 1. 添加自定义头部内容

创建`_includes/custom-head.html`文件，可以添加：
- 自定义favicon
- 额外的CSS或JavaScript
- 其他元标签

### 2. 启用评论功能

在`_config.yml`中配置Disqus：

```yaml
# 启用Disqus评论
disqus: your-disqus-shortname
```

### 3. 启用Google Analytics

在`_config.yml`中配置：

```yaml
# 启用Google Analytics
google_analytics: UA-000000-0
```

### 4. 添加目录（TOC）

在博客文章或页面的头部信息中添加：

```yaml
toc: true
```

## 六、常见问题与解决方案

1. **图片不显示**
   - 检查图片路径是否正确
   - 确保图片文件存在于指定位置
   - 使用相对路径（以/开头）

2. **本地服务器启动失败**
   - 检查依赖是否安装完整（运行`bundle install`）
   - 检查配置文件是否有语法错误
   - 查看错误信息并搜索解决方案

3. **GitHub Pages部署失败**
   - 检查仓库名是否为`yourusername.github.io`
   - 确保启用了正确的分支（通常是main或master）
   - 查看GitHub Actions或Pages设置中的错误信息
